# 指尖上的中国 — 技术设计文档

## 一、技术选型

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| 前端框架 | Vue 3 | 3.4+ | 课程要求，Composition API 是主流写法 |
| 构建工具 | Vite | 5.4+ | 比 Webpack 快，Vue 官方推荐 |
| 后端服务 | Supabase | — | BaaS 平台，免写后端代码，自带 PostgreSQL + REST API |
| 数据库 | PostgreSQL | 15 | Supabase 内置，支持行级安全、存储过程 |
| 客户端 SDK | @supabase/supabase-js | 2.110+ | 在浏览器直接操作数据库 |
| 部署 | GitHub Pages | — | 免费静态托管，配合 subtree 推送 |

**为什么不自己写后端？** Supabase 自动将数据库表暴露为 RESTful API，前端通过 JS SDK 查询。架构从"前端→后端→数据库"简化为"前端→数据库"，减少代码量的同时保留了完整的数据操作能力。

### 3.2 组件通信模式

```
父 → 子：Props 传递数据
子 → 父：$emit 触发事件
跨组件：通过 App.vue 作为中央枢纽
```

所有业务数据（bookings、works、notifications）存在 App.vue 中，通过 props 向下分发，子组件通过 emit 向上通知操作完成，App.vue 调用 api.js 后刷新数据。

### 3.3 关键文件职责

| 文件 | 职责 |
|------|------|
| `src/main.js` | 入口，创建 Vue 应用并挂载 |
| `src/App.vue` | 根组件：页面切换、状态管理、数据加载 |
| `src/api.js` | 数据层：封装所有 Supabase 调用，字段名映射 |
| `src/data.js` | 静态数据：9 类非遗信息、时间段、头像列表 |
| `src/style.css` | 全局样式：响应式布局、动画、管理后台样式 |


### 4.2 表关系总结

| 关系 | 说明 |
|------|------|
| users → bookings | 一对多：一个用户多条预约 |
| users → works | 一对多：一个用户多个作品 |
| users → interests | 一对多：一个用户多条投票 |
| ich_content.cat_id | 关联非遗分类，一个分类多条内容 |
| notifications.to_user | 可空，null 表示全站广播 |

### 4.3 关键设计决策

- **ID 生成**：预约和作品用字符串 ID（BK/WK + 时间戳36进制），不用自增 ID，便于在前端生成后直接插入
- **密码明文**：课程项目阶段，生产环境需加 bcrypt 哈希
- **图片存储**：Base64 存文本字段（压缩到 500px），不用 Supabase Storage，减少依赖
- **投票去重**：数据库层 `UNIQUE(ich_name, username)` 约束 + 前端二次校验

## 六、部署方案

### 6.1 部署架构

```
本地开发                 GitHub                  用户访问
   │                       │                       │
   │──git push────────────→│                       │
   │                       │──GitHub Actions       │
   │                       │  (可选 CI)            │
   │                       │                       │
   │──npm run build───────→│                       │
   │──git subtree push────→│                       │
   │  (dist/ → gh-pages)   │                       │
   │                       │←──GitHub Pages────────│
   │                       │  https://xxx.github   │
   │                       │  .io/Ich-app/         │
```

### 6.2 部署命令

```bash
npm run build                          # Vite 打包到 dist/
git add dist -f && git commit -m "deploy"
git subtree push --prefix dist origin gh-pages   # 推送构建产物到 gh-pages 分支
```

### 6.3 配置要点

- `vite.config.js` 中 `base: '/Ich-app/'` 确保资源路径正确
- GitHub Pages 设置指向 `gh-pages` 分支
- `.gitignore` 不能忽略 `dist/` 目录

## 七、安全策略

| 层面 | 措施 |
|------|------|
| 管理员权限 | `requireAdmin()` 从 localStorage 读取 is_admin，前端拦截 |
| 数据隔离 | 查询时用 `username` 过滤，用户只看自己的预约和作品 |
| 投票去重 | 数据库 UNIQUE 约束 + RPC 函数统计 |
| 密码安全 | 当前为明文（课程项目阶段），生产需升级为 bcrypt + Row Level Security |
